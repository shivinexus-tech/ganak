# Ganak — Open backlog acceptance register

Owner-approved structure, 2026-07-21. This register consolidates duplicate parent,
epic and sub-item checkboxes in `plans/backlog.md` without dropping deliverables.
Within each priority, work is ordered from heavy to light.

## Universal definition of done

Unless a row explicitly says docs/research only, **all** product work is done only
when: the complete scoped journey works in Hindi and English; location, timezone,
date and language survive navigation/reload without browser storage; errors are
visible and recoverable; phone and desktop browser smoke tests pass with no console
errors or horizontal overflow; answer-before-data and the design tokens are used;
the relevant permanent regression gate is added and proven non-vacuous; every
canonical gate plus production build passes; old names/routes/orphans are absent;
task-log and backlog evidence are updated; changes are reviewed, committed, pushed
and, for a go-live item, verified on production. Religious, predictive, legal and
medical claims also require cited sources and clearly stated conventions/limits.

### Owner approval is task-specific

Owner approval is required only when an individual backlog row explicitly says so.
For P0-PRASHNA-249-RESEARCH, research and KSK verification come first, then the
owner approves the method before coding begins. Do not generalise that checkpoint,
a bug-bash duration, or a separate production-approval gate to unrelated rows.

Percentages are rounded implementation estimates, not confidence scores. AI time is
focused execution time and excludes external waiting.
